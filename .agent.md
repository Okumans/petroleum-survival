# Asset Integration Agent

Use this agent when the task is to add a new asset to the game and wire it into runtime loading.

## Scope

- Inspect the relevant asset folder first:
  - Models and animation sources: `assets/objects/<asset_name>/...`
  - Textures: `assets/textures/<category>/<asset_name>/...`
- Register the asset in code only after confirming the on-disk path.
- Prefer small, focused edits that preserve the existing loading queue pattern.

## Required workflow

1. Confirm the asset path and runtime path.
2. Add the asset to `App::_setupResources()` using the existing loading task queue pattern.
3. Register the asset identifier in the correct manager:
   - `ModelManager` uses `ModelName` in `src/resource/model_manager.hpp`
   - `AnimationManager` uses `AnimationName` in `src/resource/animation_manager.hpp`
   - `MaterialManager` is string-keyed, so add or load the material under the appropriate name instead of adding an enum
4. Use `ASSETS_PATH` and match the on-disk folder structure.
5. Retrieve runtime assets through the manager APIs instead of constructing them ad hoc in gameplay code.

## Loading rules

- Use the helper lambdas in `App::_setupResources()` for models and animations.
- Keep loading inside the task queue instead of inlining duplicated setup logic.
- Make sure model dependencies exist before adding animations that reference them.
- If a new enum entry is added, make sure the matching load task exists before gameplay starts.

## Style and implementation rules

- Follow the repository naming conventions in `AGENT.md`.
- Preserve the existing `snake_case` local naming and `camelCase` public method naming.
- Keep the change minimal and local to asset setup and manager registration.
- Prefer manager-backed access over direct ownership in gameplay code.

## Validation

- Check the touched files for consistency after the edit.
- Validate that the new manager entry is loaded from `App::_setupResources()`.
- If the task changes a manager enum, verify all call sites and loading tasks are updated together.
